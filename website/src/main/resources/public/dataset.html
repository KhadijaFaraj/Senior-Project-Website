Dataset Characteristics

This dataset focuses on identifying Usage Gaps within retail organizations. The model was trained using a pruned J48 Decision Tree to predict whether an organization suffers from an ERP implementation gap.

Feature Metadata

The following attributes were extracted directly from the T1.model file:

Attribute Name Values / Categories Description
ORG_TYPE Pharmacy Supermarket Electronics Clothing Other The specific sector of the retail organization.
POSITION Staff Supervisor Manager Senior_Mgt The organizational level of the respondent.
USG_PERIOD <6m 6m_1y 1_2y 2_5y >5y Total time the ERP system has been in active use.
ERP_SYS SAP Oracle Microsoft_Dyn Odoo Other The ERP vendor solution currently implemented.
TRAINING High Low Perceived quality and availability of system training.
COMPLEXITY High Low The perceived difficulty level of using the ERP system.
PERFORMANCE High Low System responsiveness and operational efficiency.
MGT_SUPPORT High Low Extent of backing from leadership for ERP adoption.
EXTERNAL_TOOLS High Low Level of reliance on non-ERP tools (Excel, Paper).
MONITOR High Low Degree of oversight regarding system usage.
GAP (Class) Gap No_Gap Target Variable: Indicates if a usage gap exists.

Technical Model Context

The model was processed using several filters, including NumericToNominal and RenameNominalValues, to ensure all behavioral metrics were standardized into High/Low categories for better decision-tree branching.
